Technical Implementation Details

IR Signal Processing

The IR receiver captures NEC protocol signals from standard remote controls. Our firmware decodes these signals and compares them against stored authorized codes.

Lock Mechanism

A 12V solenoid lock is controlled through a transistor circuit with flyback diode protection. The CC3200 triggers the lock via GPIO with PWM for power management.

Tamper Detection

An ADXL345 accelerometer communicates with the CC3200 over I2C protocol. It's calibrated to detect sudden movements or orientation changes that might indicate tampering.

Cloud Connectivity

The system connects to AWS IoT Core using MQTT protocol over TLS/SSL. X.509 certificates are used for authentication, and JSON messages carry status updates and commands.

Key Code Highlights

IR Signal Processing


// IR signal decoding function
void processIRSignal(uint32_t irCode) {
    // Check if the code matches any authorized codes
    for(int i = 0; i < NUM_AUTHORIZED_CODES; i++) {
        if(irCode == authorizedCodes[i]) {
            // Authentication successful, unlock the door
            unlockDoor();
            return;
        }
    }
    // Authentication failed
    indicateFailure();
}

// Function to publish locker status to AWS IoT
void publishStatusToAWS(LockerStatus status) {
    char jsonBuffer[100];
    
    // Create JSON payload
    sprintf(jsonBuffer, 
            "{\"deviceId\":\"%s\",\"status\":\"%s\",\"timestamp\":%lu}", 
            DEVICE_ID, 
            statusToString(status), 
            getTimestamp());
    
    // Publish to AWS IoT topic
    MQTTClient_publishMessage(client, "lockers/status", 
                             &jsonPayload, &deliveryToken);
}
