Smart Locker System

A secure and intelligent approach to personal storage using the CC3200 microcontroller

View Demo

Project Overview

In today's world, securing personal belongings in shared spaces is a growing concern. Our Smart Locker System provides a secure, efficient, and modernized approach to personal storage using the CC3200 microcontroller. This system enables users to lock and unlock a storage unit using an IR remote, ensuring seamless authentication and access. To add a connected element, the locker system integrates with AWS IoT to track usage and provide real-time status updates.

Secure Authentication

Uses IR remote signals for secure access authentication with error handling for invalid attempts.

Fast Response

Quick unlock mechanism with automatic relocking after a set duration for added security.

AWS IoT Integration

Real-time monitoring and status updates through cloud connectivity.

Design

Functional Specification

The smart locker operates on a state-based model:

  • Idle State: The locker remains locked and waits for a remote input.
  • Authentication State: The system verifies the received signal from an IR remote against stored valid signals.
  • Unlock State: If authentication succeeds, the locker unlocks for a set duration before relocking automatically.
  • Error State: If authentication fails, the locker remains locked and provides an error response.

System Architecture

Our system comprises these core components:

  • CC3200 Microcontroller: The core component managing authentication, communication, and control.
  • IR Receiver: Reads signals from an IR remote for authentication.
  • Solenoid Lock: Controls the physical locking mechanism.
  • Accelerometer: Detects unauthorized movements or tampering attempts.
  • AWS IoT Integration: Logs locker status (locked/unlocked/tampered) and allows remote monitoring.

Implementation

The Smart Locker System is built using the CC3200 LaunchPad as the primary controller. The system uses various communication protocols to interface with different components:

Hardware Implementation

The physical locker mechanism consists of a solenoid lock controlled by the CC3200 through a transistor circuit. The IR receiver is connected to GPIO pins for signal processing, while the accelerometer communicates via I2C protocol to detect tampering attempts.

Software Implementation

The firmware is structured around a state machine that handles authentication, locking/unlocking, and error conditions. The CC3200's WiFi capabilities are utilized to connect to AWS IoT, where MQTT protocol is used to publish status updates and receive commands if needed.

Challenges & Solutions

IR Signal Decoding

Challenge: Accurately decoding IR signals from different remote controls.

Solution: Implemented a robust IR library with error checking and signal validation.

Power Management

Challenge: Ensuring the locking mechanism operates reliably with limited power.

Solution: Designed an efficient power control circuit with capacitor-based energy storage.

Cloud Connectivity

Challenge: Maintaining stable AWS IoT connection with minimal latency.

Solution: Implemented connection recovery mechanisms and local caching of critical operations.

Future Work
